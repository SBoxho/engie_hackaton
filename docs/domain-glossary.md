# Domain Glossary

Date: 2026-06-18

## Product Flow

| Term | Meaning in this repository |
| --- | --- |
| Observe | Show the current or replayed electricity state: demand, generation, regional context, weather, official signal, provenance. |
| Forecast | Estimate future demand and pressure over selected horizons with uncertainty and source routing. |
| Explain | Show why an observed or forecast signal is high/low using transparent drivers, model explanations, and data provenance. |
| Simulate | Change an assumption and compare a baseline outlook to a scenario outlook. |
| Digital twin | A stateful analytical representation of the electricity-demand system with observed state, forecast state, explanations, scenario assumptions, and validation metadata. The current repo is a prototype, not a complete digital twin. |

## Data Modes

| Term | Meaning |
| --- | --- |
| Live mode | Fetches or reads current public data and caches it locally. Controlled by `APP_MODE=live`. |
| Demo mode | Default offline mode using tracked `demo_data/` artifacts. Controlled by `APP_MODE=demo`. |
| Replay mode | UI label for historical demo/cached data shown as replay, not live. Demo data uses a fixed historical window from `demo_data/manifest.json`. |
| Cached snapshot | Previously fetched public data stored under `data/raw/` or `data/processed/`. |
| Synthetic value | A value generated by code rather than measured directly from a public source. |
| Hard-coded assumption | A fixed coefficient, threshold, or preset embedded in code or config. |
| Mock data | Test-only data fixtures. In the target architecture, mocks must stay in tests. |

## Electricity System Terms

| Term | Meaning |
| --- | --- |
| Demand | Electricity wanted/consumed by users, represented as `consumption_mw`. |
| Load | Synonym for demand in many power-system contexts. |
| Generation | Electricity produced domestically by source, such as nuclear, wind, solar, hydro, gas, coal, oil, and bioenergy. |
| Production | Synonym for generation in this app. |
| Total production | Sum of generation sources in MW. |
| Renewable production | Wind, solar, hydro, and bioenergy production in MW. |
| Fossil production | Gas, coal, and oil production in MW. |
| Renewable share | Renewable production divided by total production. |
| Fossil share | Fossil production divided by total production. |
| Imports | Imported electricity when exchanges are positive toward France. Current import mix is not decomposed by foreign generation source. |
| Exports | Exported electricity when exchanges flow out of France. |
| Net imports | Physical exchange balance normalized by the cleaning pipeline. |
| Supply margin | Available domestic production plus imports minus demand. |
| Import requirement | Demand minus domestic production when demand exceeds domestic production. |
| CO2 intensity | Grams of CO2 per kWh reported by source data or derived artifact. |
| EcoWatt | Official public signal describing electricity-system tension. In this repo, ODRE history is no-key; live API token is optional. |

## Forecasting Terms

| Term | Meaning |
| --- | --- |
| Forecast origin | The timestamp when a forecast is made, using only data available at or before that time. |
| Horizon | How far into the future a forecast target is from the origin, such as 1h, 3h, 6h, 24h, or 48h. |
| Target timestamp | The timestamp being forecast. |
| Nowcast | A very near-term forecast, currently used for 0 to 3 hours when model points are validated. |
| Baseline | A transparent reference rule used for comparison, such as persistence, previous-day, or previous-week. |
| Persistence forecast | Predicts future demand from the latest observed demand. |
| Previous-day baseline | Predicts a target using the same local time from the previous day. |
| Previous-week baseline | Predicts a target using the same local time from the previous week. |
| p10 | Lower forecast quantile; roughly a 10th percentile estimate when properly calibrated. |
| p50 | Median forecast estimate. |
| p90 | Upper forecast quantile; roughly a 90th percentile estimate when properly calibrated. |
| Prediction interval | The range between p10 and p90 in the public chart. |
| Backtest error | Error measured on historical forecast origins and targets. |
| Forecast route | The source path for a forecast point: official, model, baseline, fallback, replay, or unsupported. |
| Unsupported horizon | A future hour shown without a recommendation because no validated public forecast/model route exists. |

## Explainability Terms

| Term | Meaning |
| --- | --- |
| Driver | A human-readable factor that contributes to observed or forecast pressure, such as demand, supply margin, weather, or uncertainty. |
| Feature | A model input, such as lagged demand, calendar variables, holiday flags, or weather. |
| Feature family | A grouped set of related features used for explanation, such as recent demand, weekly pattern, calendar, weather, or RTE forecast. |
| Ablation explanation | Approximate method that changes/removes feature groups and observes prediction sensitivity. Current model explanations use this style. |
| SHAP | A model-explanation method for attributing prediction contributions. It is a future target, not currently implemented as the public explainability layer. |
| Causal explanation | A claim that one factor caused another. The current app should not make causal claims from model associations. |

## Simulation Terms

| Term | Meaning |
| --- | --- |
| Scenario preset | A predefined WHAT IF? assumption, such as cold snap, low wind, generation outage, EV shift, or solar above forecast. |
| Baseline scenario | The unmodified forecast outlook used for comparison. |
| Modified scenario | The outlook after applying the selected assumption. |
| Scenario coefficient | A fixed value used to translate an assumption into MW or CO2 changes. Current coefficients are hard-coded and directional. |
| Directional simulation | A public-decision aid that indicates direction and relative magnitude, not dispatch, tariff, or verified emissions accounting. |
| Assumption artifact | Future versioned file that should store scenario coefficients, labels, units, and public-source citations. |

## Quality and Governance Terms

| Term | Meaning |
| --- | --- |
| Provenance | Source URL, dataset ID, retrieval time, artifact generation time, and transformation notes. |
| Freshness | How recent the latest data point or artifact is relative to now or to a replay clock. |
| Leakage | Use of future information when training, evaluating, or explaining forecasts. The target architecture must avoid it. |
| Promotion gate | A rule that decides whether a model/horizon is allowed in the public forecast route. |
| Drift | Change in schema, source availability, data distribution, forecast error, or explanation behavior over time. |
| Quality finding | A structured warning/error from schema, cadence, missingness, freshness, duplicate, or physical-balance checks. |
